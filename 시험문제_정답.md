## Part 1. 객관식 정답 및 해설

### Q1. 정답: **(B)**

`@tool` 데코레이터는 **독립 함수**(standalone function)에 적용해야 한다. 클래스 메서드에 쓰면 `self`가 tool 스키마에 노출되어 LLM이 혼란을 겪는다.

- (A) 클래스 메서드에 `@tool` → `self`가 파라미터로 노출됨
- (B) 독립 함수에 `@tool` → 올바른 사용법
- (C) `tool(search_hospital)` 후위 호출도 동작하지만 관용적이지 않음
- (D) `__init__` 내부 정의는 동작하지만 매 인스턴스마다 새 tool이 생성되어 비효율적

**과제 코드**: `tools.py`에서 `@tool`을 독립 함수에 올바르게 사용하고 있음.

## Part 2. 단답 / 코드 교정 정답 및 해설

### Q11. 코드 교정

**Bug 1**: `MemorySaver` import 누락. `from langgraph.checkpoint.memory import MemorySaver` 추가 필요.

**Bug 2**: `@tool`을 클래스 메서드(`self` 포함)에 적용. `self`가 tool 스키마에 노출됨. 독립 함수로 분리하거나 `@tool` 제거 후 `StructuredTool.from_function()`으로 래핑해야 함.

**Bug 3**: `response_fromat`은 오타(`format` → `fromat`). 또한 `ToolStrategy`도 import되지 않음. 올바르게는 `response_format=ToolStrategy(ChatResponse)`이되, `ToolStrategy` import도 필요.

---

### Q12. 빈칸 채우기

> `create_agent()`의 `system_prompt` 파라미터는 **`str`** 또는 **`SystemMessage`** 타입을 받을 수 있다.
>
> 멀티턴 대화를 유지하려면 `create_agent()`에 `checkpointer=**MemorySaver**()`와 같이 서버 전체에서 단 한 번 생성된 인스턴스를 전달해야 한다.

### Q13. 설계 오류 설명

**문제점**: `process_query()`가 호출될 때마다 `InMemorySaver()`를 새로 생성한다. 새 체크포인터에는 이전 대화 기록이 없으므로 `thread_id`를 전달해도 **대화 연속성이 유지되지 않는다**. `create_agent()`도 매번 재생성되어 비효율적이다.

**수정 코드**:

```python
class AgentService:
    def __init__(self):
        self._checkpointer = InMemorySaver()  # 한 번만 생성
        self._agent = None

    def _get_agent(self):
        if self._agent is None:
            self._agent = create_agent(
                model=self.model,
                tools=self.tools,
                checkpointer=self._checkpointer
            )
        return self._agent

    def process_query(self, query: str, thread_id: str) -> str:
        agent = self._get_agent()
        result = agent.invoke(
            {"messages": [{"role": "user", "content": query}]},
            {"configurable": {"thread_id": thread_id}}
        )
        return result["messages"][-1].content
```

**핵심**: checkpointer는 `__init__`에서 1회 생성, agent는 캐싱하여 재사용.

> 이 문제는 과제 코드의 수정 전 `agent_service.py`와 완전히 동일한 버그 패턴이며, 현재 코드에서는 수정 완료됨.

### Q15. 자유 서술

`response_format=ToolStrategy(ChatResponse)`는 LangGraph의 공식 API로, **에이전트의 최종 응답 형식을 Pydantic 모델로 강제**한다. 에이전트가 마지막에 반드시 `ChatResponse` 스키마에 맞는 구조화된 출력을 생성하도록 보장하며, 중간 tool 호출과 최종 응답을 명확히 구분할 수 있다.

반면 `ChatResponse`를 `@tool`로 직접 정의하는 방식은 LLM이 해당 tool을 **선택적으로** 호출하므로, 반드시 호출된다는 보장이 없다. 또한 tool 호출 결과가 다시 LLM에 피드백되어 추가 처리가 필요할 수 있어 비효율적이다.

공식 API 관점에서 `response_format=ToolStrategy(ChatResponse)` 방식이 권장된다. 응답 형식이 프레임워크 레벨에서 강제되므로 일관성이 보장되고, tool 목록이 깔끔하게 유지되기 때문이다.
